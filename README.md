### 프로젝트 실행 방법

#### 요구 사항
- Node.js **18 이상** 권장 (Vite 사용)
- npm

#### 설치

```bash
npm install
```

#### 개발 서버 실행

```bash
npm run dev
```

#### 프로덕션 빌드

```bash
npm run build
```

#### 빌드 결과 미리보기

```bash
npm run preview
```

### 구현한 기능 설명

- **현재 위치 기반 조회**
  - 브라우저 Geolocation(`navigator.geolocation`)으로 좌표를 가져오고, 해당 좌표의 날씨를 표시합니다.
  - 좌표를 사람이 읽기 좋은 이름으로 보여주기 위해 역지오코딩을 사용합니다.

- **장소 검색**
  - 행정구역 데이터 기반 자동완성으로 검색 UX를 제공하고, 선택한 장소의 좌표를 얻어 날씨를 조회합니다.

- **날씨 요약 + 시간대별 기온(오늘 00~23시)**
  - 현재 기온/최저/최고 및 오늘 시간대별 기온을 표시합니다.

- **즐겨찾기(최대 6개)**
  - 즐겨찾기 추가/해제(★ 토글)
  - 즐겨찾기 목록 표시(각 항목별 요약 날씨 병렬 조회)
  - **이름 수정**(인라인 편집: 수정/저장/취소)
  - 삭제
  - 즐겨찾기는 브라우저 **localStorage**에 저장됩니다.

- **테스트용 UI 강제(디버그)**
  - 검색한 장소의 “정보가 제공되지 않습니다” 상태를 확인하기 위해 URL 파라미터를 지원합니다.
  - 예: `/?debugSearchWeather=empty`
  - 사용 방법: 위 파라미터를 붙인 URL로 접속한 뒤, **아무 장소나 검색**하면 검색한 장소 영역이 강제로 *정보 없음* 상태로 표시됩니다.

### 기술적 의사결정 및 이유

#### 번들러로 Vite를 선택한 이유
- **빠른 개발 경험**: dev 서버가 빠르고 HMR이 안정적이라 UI 반복 작업(레이아웃/상태/스타일) 속도가 좋습니다.
- **설정이 단순함**: React 앱 구성에 필요한 플러그인만으로 깔끔하게 운영할 수 있습니다.
- **배포 친화적**: 정적 빌드 산출물(`vite build`)이 명확해서 Vercel 같은 호스팅에 올리기 쉽습니다.

#### 외부 날씨 API로 Open-Meteo를 사용한 이유
- **키 없이 시작 가능**: 빠르게 기능을 검증하기에 적합합니다(초기 세팅/보안 부담이 적음).
- **필요한 데이터 형태가 맞음**: 현재/일별(최저·최고)/시간대별 데이터를 한 API로 가져올 수 있어 요구사항을 단순하게 구현할 수 있습니다.
- **클라이언트 직접 호출에 적합**: 프로토타입/개인 프로젝트에서 별도 백엔드 없이도 구성하기 쉽습니다.

#### 지오코딩/역지오코딩으로 OpenStreetMap Nominatim을 사용한 이유
- **키 없이 사용 가능**: 검색(지오코딩)과 현재 위치 이름 변환(역지오코딩)을 빠르게 붙일 수 있습니다.
- **주소 기반 UX 구현**: 사용자는 좌표가 아니라 “장소명/주소”로 인지하므로, 날씨 조회 흐름에 필수적인 보조 기능입니다.
- **로컬라이징**: `accept-language=ko` 등으로 한국어 표시를 우선하도록 구성할 수 있습니다.

### 사용한 기술 스택 정리

- **프론트엔드**
  - React, TypeScript
  - Tailwind CSS
  - React Router (`react-router-dom`): 페이지 라우팅
  - TanStack Query (`@tanstack/react-query`): 데이터 패칭/캐싱/병렬 조회

- **빌드/도구**
  - Vite
  - ESLint
  - PostCSS / Autoprefixer

- **외부 API**
  - Open-Meteo Weather API
  - OpenStreetMap Nominatim (geocoding/reverse geocoding)

- **브라우저 API / 저장소**
  - Geolocation API
  - localStorage (즐겨찾기 저장)
